什麼是AJAX

AJAX是一種撰寫JavaScript的技巧，可以在不更新網頁或重新載入的情況下，及時從後端伺服器取得資料的方法。常見的情境有： - 在註冊帳號的頁面檢查與判斷使用者輸入資料是否與資料庫中已存在的使用者重複 - 電商網站的消費者將商品加入購物車時，即時與資料庫確認商品庫存是否足夠等等 - 在google搜尋中輸入關鍵字時，及時跳出來的搜尋建議

這裡我們先將注意力放在readyState狀態上，其狀態代碼從0-4份別代表的意義如下 - 0 - 你已經產生了一個XMLHttpRequest，但是還沒有連結要撈取的資料 - 1 - 你觸發了open()，但還沒有把資料傳送到伺服器 - 2 - 偵測到你有用send() - 3 - loading... - 4 - 你撈到資料了，已經完全接收

由於在這裡我們使用的資料量相當的少，所以在send()之後，readyState狀態從2到4的時間將當的短，所以在console查看時，會覺得readyState狀態直接從1跳到4了。

AJAX同步觀念

前面在open()的三個參數中，第三個參數同步或非同步又是什麼呢？

- true 非同步，不等待資料回傳，就直接讓程式碼繼續往下執行
- false 同步，它會等資料傳回來，接收到資料回傳後才讓程式碼繼續往下執行

由於後端資料庫的資料量可能會相當龐大，因此一般在撰寫AJAX時，較常會採用非同步的方式撰寫。

但是採用非同步的時候，會遇到一個問題，因為在資料尚未回傳程式碼就繼續往下執行了，那我們要如何取得後端回傳的資料呢？ 這個時候可以採用XMLHttpRequest()中.onload這個屬性，因為只有當readyState狀態是4的時候，也就是確認資料有回傳了onload才不會是null，當onload不為null時，我們就可以執行function()來取得回傳的資料。 因此我們可以用下面的範例的方法來取的前面範例中回傳的遠端json檔案資料

簡單來說，AJAX是用以下的四個步驟來達到再不更新網頁的前提下，與後端伺服器或遠端伺服器的互動 1. 建立一個XMLHttpRequest() 2. 傳送資料到對方伺服器 3. 回傳資料到自己的瀏覽器 4. 拿到資料後再決定要如何透過程式碼處理收到的資料

##　HTTP狀態碼 在瀏覽器的開發者模式中，在Nework頁面可以看到網頁、圖片或是檔案等等的資訊，其中有一個就是status狀態碼，而不同的status狀態碼又代表什麼呢？常見的狀態碼有： - 200 ：資料有正確回傳， 有撈到資料 - 404 ：資料讀取錯誤，沒有撈到資料

而在XMLHttpRequest()中，我們也可以看到有status屬性，因此可以藉由status的值來判斷撈到的資料是否正確。在這裡可以在responseText獲得的資料是JSON格式的字串，因此就可以用JSON.parse的方法來將字串轉換為物件，來讀取並顯示在網頁中。

還有更多關於HTTP狀態碼的說明，可以參閱網頁開發人員應了解的 HTTP 狀態碼。

Cross-Origin Resource Sharing (CORS)？

在我們使用XMLHttpRequest()方法時，還有一件重要的事情是我們希望撈取的資料，對方有沒有開啟跨網域的功能，也就是所謂的Cross-Origin Resource Sharing (CORS)，如果對方沒有開啟CORS，那就無法使用XMLHttpRequest()方法區發一個請求給對方，然後將資料撈回來。

想知道網站是否有開啟COR可以到test-cors.org，把網址丟到裡面做測試，若status是200就可以使用XMLHttpRequest()方法獲取資料；若status是0，就代表該網站無法以XMLHttpRequest()方法獲取資料。
